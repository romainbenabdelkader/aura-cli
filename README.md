AURA CLI (reference demo)

Not related to Neo4j ‚ÄúAura CLI‚Äù.

This repository contains a minimal command-line tool demonstrating the core concepts of the AURA (European Origin Proof) - Public Draft standard.

It issues and verifies a signed AURA manifest for a local digital asset (audio, image, text, video) using cryptographic hashes and Ed25519 signatures.

‚∏ª

Reference demo only

This CLI is a reference demonstration of the AURA draft.

It is not production-ready and does not implement a full trust registry (TPKR).

It does not perform DRM, watermarking, fingerprinting, similarity detection, or content recognition.

‚∏ª

What this demo illustrates
‚Ä¢ Hashing a local asset (SHA3-256)
‚Ä¢ Issuing a signed AURA manifest (JSON)
‚Ä¢ Verifying asset integrity and manifest signature
‚Ä¢ Expressing a simple rights signal (TDM opt-out)
‚Ä¢ Offline, portable proof of origin (no platform dependency)

‚∏ª

Demo scenario (as shown in the video)
	1.	Verify an original asset against its AURA manifest ‚Üí VALID
	2.	Make a byte-for-byte copy of the asset ‚Üí still VALID
	3.	Alter the copied file (even by 1 byte)
	4.	Verify again ‚Üí FAIL (asset hash mismatch)

This demonstrates that the AURA proof is strictly bound to the binary content of the asset.

Requirements
pip install -r requirements.txt

Quick start

1. Initialise a local issuer (demo keys)
python aura_cli.py init
2. Issue an AURA manifest
python aura_cli.py issue --asset example.wav --asset-type audio --tdm-opt-out
This generates a .aura manifest file next to the asset.

3. Verify an AURA manifest
python aura_cli.py verify --asset example.wav --manifest example.wav.aura

The issuer public key is embedded in the manifest, allowing independent verification without access to the issuer environment.

CLI reference

Initialise issuer keys:
python aura_cli.py init
Issue:
python aura_cli.py issue --asset PATH --asset-type audio|image|text|video --manifest OUTPUT.aura --tdm-opt-out
Verify:
python aura_cli.py verify --asset PATH --manifest PATH.aura
Files generated by this demo
‚Ä¢ issuer_private.pem (demo private key)
‚Ä¢ issuer.json (demo issuer metadata)
‚Ä¢ *.aura (AURA manifests containing the embedded public key)
