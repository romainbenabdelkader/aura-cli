Context and purpose

AURA CLI is a minimal, open-source reference implementation designed to support experimentation, discussion and evaluation around proof of origin, provenance and consent documentation for digital assets in the context of AI systems.

It is developed as a testbed artefact, aligned with European public policy objectives, including:

‚Ä¢	copyright and TDM opt-out frameworks,

What this testbed demonstrates

The CLI demonstrates how a digital asset can be associated with a verifiable proof of origin through:
	
‚Ä¢	cryptographic hashing of the asset,
	
‚Ä¢	deterministic canonicalisation of a manifest,
	
‚Ä¢	Ed25519 digital signatures,
	
‚Ä¢	issuer identification and resolution via a registry (TPKR demo),
	
‚Ä¢	explicit declaration of rights-related signals (e.g. TDM opt-out).

Trust model (testbed)

This implementation uses a local Trust Public Key Registry (TPKR) for demonstration purposes.

Key characteristics:
	
‚Ä¢	issuers are identified by a cryptographic fingerprint,
	
‚Ä¢	public keys are resolved via a registry,
	
‚Ä¢	the registry is not governed in this demo.

This reflects a testbed assumption.

In a production or EU-scale deployment:
	
‚Ä¢	issuer onboarding would follow governance rules,
	
‚Ä¢	registry state would be governed,
	
‚Ä¢	registry hashes could be published or anchored (e.g. public bulletin, ledger, or EU trust infrastructure).

The trust model is intentionally explicit and inspectable, not implicit.
