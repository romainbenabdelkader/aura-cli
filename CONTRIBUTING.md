# Contributing

aura-cli is a non-normative demonstrator CLI for the AURA proof-of-origin model.

Contributions are welcome via GitHub Issues and Pull Requests.

## Scope

In scope:
- clarity and correctness of the demonstrator (init / issue / create / verify)
- alignment with the AURA standard (SHA3-256 asset hashing, Ed25519 signatures)
- documentation and examples

Out of scope (see NON_GOALS.md):
- production deployment, DRM, watermarking, fingerprinting, monitoring or
  enforcement

## How to contribute

- Open an Issue to discuss a change or report a bug
- Submit a focused Pull Request
- Verify the demo still passes: `bash demo_integrity.sh`
